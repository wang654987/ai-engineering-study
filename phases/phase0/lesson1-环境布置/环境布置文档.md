## Step 2: Python + uv

### 安装 uv
uv 是 Rust 编写的 Python 包管理器，比 pip 快 10-100 倍，支持全局缓存和硬链接（多虚拟环境不重复占硬盘）。

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 安装 Python 3.12
```bash
uv python install 3.12
```

### 创建虚拟环境
```bash
uv venv
source .venv/bin/activate      # Linux/WSL
# .venv\Scripts\activate       # Windows
```

### 安装基础科学计算包
```bash
uv pip install numpy matplotlib jupyter
```

### 验证
```python
import sys
print(f"Python {sys.version}")

import numpy as np
print(f"NumPy {np.__version__}")
a = np.array([1, 2, 3])
print(f"Vector: {a}, dot product: {np.dot(a, a)}")  # 期望 14
```

**当前版本：**
- Python 3.12.13
- NumPy 2.4.6
- Matplotlib 3.10.9
- Jupyter 4.5.8

### 关键概念
| | pip | uv |
|------|------|------|
| 语言 | Python（解释型） | Rust（编译型） |
| 下载 | 串行 | 并行 |
| 硬盘 | 每个虚拟环境独立复制 | 全局缓存 + 硬链接 |
| 速度 | 基准 | 快 10-100 倍 |

## Step 4: Rust

### 安装 rustup (Rust 工具链管理器)
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```
安装过程中选 1（默认），装完执行：
```bash
source "$HOME/.cargo/env"
```

### 验证
```bash
rustc --version
cargo --version
```

**当前版本：** rustc 1.96.0, cargo 1.96.0

### 三个工具职责
| 工具 | 职责 | 类比 Python |
|------|------|------|
| rustup | 管理 Rust 工具链/版本 | uv 管 Python 版本 |
| rustc | 编译源码 → 机器码 | python 运行 .py |
| cargo | 包管理 + 工程构建 | pip + 项目脚手架 |

### Rust vs Python 运行方式
| | Rust（编译型） | Python（解释型） |
|------|------|------|
| 运行前 | 必须编译成二进制 | 直接跑 |
| 产物 | 独立可执行文件，搬到其他机器直接跑 | 需要 Python 解释器陪着 |

### Julia 的 JIT 编译
Julia 使用 **JIT（Just-In-Time）编译**：第一次运行代码时即时编译成机器码，之后再跑直接用编译好的版本。

| 语言 | 运行方式 | 速度 |
|------|------|------|
| Python | 解释执行（边跑边翻译） | 慢 |
| Julia | JIT 编译（第一次时编译） | 快 |
| Rust | AOT 编译（提前全编译） | 快 |


**当前设备：** RTX 4060 Laptop，驱动 610.43.02，CUDA 13.3

### 安装 PyTorch (CUDA 版本)
```bash
uv pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
```

> CUDA 不是编程语言，是 NVIDIA 的计算平台——让程序能调用 GPU 加速。

### 验证
```python
import torch
print(f"CUDA available: {torch.cuda.is_available()}")
if torch.cuda.is_available():
    print(f"GPU: {torch.cuda.get_device_name(0)}")
```

**当前版本：** PyTorch 2.6.0+cu124, CUDA True

### 全局缓存机制
uv 下载的包存储在 `~/.cache/uv/`（全局缓存），各虚拟环境通过硬链接指向缓存，不重复占用硬盘：

```
pip 方式：   每个虚拟环境各复制一份 700MB → N × 700MB
uv 方式：    缓存一份 700MB + 各虚拟环境硬链接（0MB 额外）
```

删除虚拟环境不会删除缓存。换 CUDA 版本（如 cu130 → cu124）会重新下载。

### 练习 3：四语言 Hello World

| 语言 | 命令 | 类型 |
|------|------|------|
| Python | `python -c "print('Hello, World!')"` | 解释型 |
| Node.js | `node -e "console.log('Hello, World!')"` | 解释型/JIT |
| Julia | `julia -e 'println("Hello, World!")'` | JIT 编译 |
| Rust | 写 `.rs` → `rustc` 编译 → 运行二进制 | 编译型 |

Rust 完整示例：
```bash
mkdir -p /tmp/hello
echo 'fn main() { println!("Hello, World!"); }' > /tmp/hello/main.rs
rustc /tmp/hello/main.rs -o /tmp/hello/hello
/tmp/hello/hello
```

### `-e` / `-c` 参数含义
| 语言 | 参数 | 含义 |
|------|------|------|
| Python | `-c` | command |
| Node.js | `-e` | evaluate |
| Julia | `-e` | eval |

都表示"直接在命令行执行后面的字符串，不跑文件"。

## 知识测验（Phase 0 自测）

### 第 1 题（课前）

> Why do AI projects need a separate virtual environment?
> 为什么 AI 项目需要单独的虚拟环境？

- A. Virtual environments make code run faster / 虚拟环境让代码跑得更快
- B. They isolate dependencies so different projects don't conflict / 它们隔离依赖，让不同项目不冲突
- C. Python requires virtual environments to import packages / Python 必须有虚拟环境才能导入包
- D. Virtual environments provide GPU access / 虚拟环境提供 GPU 访问

**答案：B**

解释：虚拟环境隔离每个项目的依赖版本。没有它，为一个项目升级 PyTorch 可能会破坏另一个依赖旧版本的项目。C 不正确——Python 没有虚拟环境也能 `import` 包。

### 第 2 题（课前）

> What does CUDA provide for AI workloads?
> CUDA 为 AI 工作负载提供什么？

- A. A Python package manager for ML libraries / 一个 ML 库的 Python 包管理器
- B. A web framework for deploying models / 一个部署模型的 Web 框架
- C. Parallel computing on NVIDIA GPUs for matrix operations / 在 NVIDIA GPU 上进行并行计算，处理矩阵运算
- D. A container runtime for model serving / 模型服务的容器运行时

**答案：C**

解释：CUDA 是 NVIDIA 的并行计算平台，让矩阵运算能同时在 GPU 的几千个核心上跑。PyTorch 和 TensorFlow 底层都用它。注意 `nvidia-smi` 只能查看 GPU，不能验证 PyTorch 是否能调用它。

### 第 3 题（课后）

> In the four-layer environment stack, which layer must be installed first?
> 在四层环境栈中，哪一层必须先安装？

- A. AI/ML Libraries (PyTorch, JAX) / AI/ML 库
- B. Package Managers (pip, npm, cargo) / 包管理器
- C. Language Runtimes (Python, Node.js) / 语言运行时
- D. System Foundation (OS, shell, GPU drivers) / 系统基础（OS、Shell、GPU 驱动）

**答案：D**

解释：安装顺序从下往上——先操作系统和驱动，再包管理器，再语言运行时，最后 AI 库。每层依赖下面那层。

```
层 4: AI/ML 库（PyTorch）         ← 最后装
层 3: 语言运行时（Python、Node）   ← 第三
层 2: 包管理器（uv、pnpm）        ← 第二
层 1: 系统基础（OS、驱动）        ← 最先装
```

### 第 4 题（课后）

> What is the purpose of uv in a Python AI project?
> uv 在 Python AI 项目中的作用是什么？

- A. A GPU monitoring tool / GPU 监控工具
- B. An ultra-fast Python package installer and resolver / 超快的 Python 包安装器和依赖解析器
- C. A neural network visualization library / 神经网络可视化库
- D. A CUDA compiler for custom kernels / 自定义 CUDA 内核的编译器

**答案：B**

解释：uv 是用 Rust 写的 Python 包安装器，替代 pip，依赖解析和安装速度快 10-100 倍。它还支持全局缓存和硬链接，多个虚拟环境不重复占用硬盘。

### 第 5 题（课后）

> How do you verify that PyTorch can access your GPU?
> 如何验证 PyTorch 能用 GPU？

- A. `import torch; print(torch.__version__)` — 打印 PyTorch 版本
- B. `nvidia-smi --query` — 查询系统 GPU
- C. `import torch; print(torch.cuda.is_available())` — 检查 CUDA 是否可用
- D. `python -c 'import gpu'` — 不存在这个模块

**答案：C**

解释：`torch.cuda.is_available()` 返回 `True` 才说明 PyTorch 能访问 CUDA GPU。A 只显示版本号，B 只确认系统有 GPU（不代表 PyTorch 能用）。Apple Silicon 上用 `torch.backends.mps.is_available()`。
